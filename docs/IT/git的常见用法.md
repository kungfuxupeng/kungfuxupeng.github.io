#卡片/git

以下是常见的 Git 命令及解释，按照从常见到不常用的顺序排列：

---

### 基础操作
`git init`:: 在当前目录初始化一个新的 Git 仓库
<!--SR:!2025-12-03,4,270-->
`git clone <url>`:: 克隆远程仓库到本地
<!--SR:!2025-12-02,3,250-->
`git add <file>`:: 将文件添加到暂存区（`git add .` 添加所有文件）
<!--SR:!2025-12-03,4,270-->
`git commit -m "message"`:: 提交暂存区的文件到仓库并添加描述信息
<!--SR:!2025-12-03,4,270-->
`git status`:: 查看工作区和暂存区的状态
<!--SR:!2025-12-02,3,250-->
`git push`:: 将本地提交推送到远程仓库
<!--SR:!2025-12-03,4,270-->
`git pull`:: 从远程仓库拉取更新并合并到本地（`git fetch + git merge`）
<!--SR:!2025-12-03,4,270-->
`git log`:: 查看提交历史记录
<!--SR:!2025-12-03,4,270-->

---

### 分支管理
`git branch`:: 查看所有本地分支（`-r` 查看远程分支，`-a` 查看所有分支）
<!--SR:!2025-12-02,3,250-->
`git branch <branch-name>`:: 创建新分支
<!--SR:!2025-12-02,3,250-->
`git checkout <branch-name>`:: 切换到指定分支
<!--SR:!2025-12-02,3,250-->
`git checkout -b <branch-name>`:: 创建并切换到新分支
<!--SR:!2025-12-02,3,250-->
`git merge <branch-name>`:: 将指定分支合并到当前分支
<!--SR:!2025-12-02,3,250-->
`git branch -d <branch-name>`:: 删除本地分支（`-D` 强制删除）
<!--SR:!2025-12-02,3,250-->
`git push origin --delete <branch-name>`:: 删除远程分支
<!--SR:!2025-11-30,1,230-->

---

### 撤销与恢复
`git restore <file>`:: 恢复工作区的文件到最新提交状态（Git 2.23+）
<!--SR:!2025-11-30,1,230-->
`git restore --staged <file>`:: 将文件从暂存区移回工作区
<!--SR:!2025-11-30,1,230-->
`git reset <file>`:: 将文件从暂存区撤出（同 `git restore --staged`）
<!--SR:!2025-12-02,3,250-->
`git reset --hard <commit>`:: 重置工作区和暂存区到指定提交（谨慎使用）
<!--SR:!2025-11-30,1,230-->
`git revert <commit>`:: 创建一个新提交来撤销指定提交的更改
<!--SR:!2025-11-30,1,230-->

---

### 远程仓库操作
`git remote -v`:: 查看远程仓库地址
<!--SR:!2025-12-02,3,250-->
`git remote add <name> <url>`:: 添加远程仓库
<!--SR:!2025-12-02,3,250-->
`git fetch <remote>`:: 从远程仓库下载数据但不合并
<!--SR:!2025-11-30,1,230-->
`git push -u origin <branch>`:: 推送分支并设置上游跟踪（首次推送时常用）
<!--SR:!2025-12-02,3,250-->

---

### 高级操作
`git stash`:: 临时保存工作区的修改（`git stash pop` 恢复）
<!--SR:!2025-11-30,1,230-->
`git tag <tag-name>`:: 为当前提交打标签（`-a` 创建附注标签）
<!--SR:!2025-12-02,3,250-->
`git diff`:: 查看工作区与暂存区的差异（`--staged` 查看暂存区与仓库差异）
<!--SR:!2025-11-30,1,230-->
`git rebase <branch>`:: 将当前分支变基到指定分支（重写提交历史）
<!--SR:!2025-11-30,1,230-->
`git cherry-pick <commit>`:: 将指定提交应用到当前分支
<!--SR:!2025-11-30,1,230-->

---

### 不常用但实用
`git bisect`:: 使用二分查找定位引入问题的提交
<!--SR:!2025-11-30,1,230-->
`git blame <file>`:: 查看文件的每一行最后修改信息
<!--SR:!2025-11-30,1,230-->
`git submodule`:: 管理子模块（如 `add`、`update`）
<!--SR:!2025-11-30,1,230-->
`git reflog`:: 查看本地仓库的操作记录（用于恢复误操作）
<!--SR:!2025-11-30,1,230-->
`git archive`:: 将仓库文件打包成压缩包
<!--SR:!2025-11-30,1,230-->

---

### 配置相关
`git config --global user.name "name"`:: 配置全局用户名
<!--SR:!2025-12-02,3,250-->
`git config --global user.email "email"`:: 配置全局邮箱
<!--SR:!2025-12-02,3,250-->
`git config --list`:: 查看所有配置项
<!--SR:!2025-12-02,3,250-->

---

以上命令覆盖了日常开发中 90% 以上的场景，建议优先掌握基础操作和分支管理部分。
